# Dataphin 智能化能力介绍

## 产品定位
数据系统产品，聚焦数据研发、数据治理、数据运营全面智能化升级，助力企业构建"好数据"、管好数据、用好数据，释放数据价值。

## 核心智能化能力

### 一、智能研发
告别重复劳动，专注于价值创造。数据研发效率提升70%。

#### 1. 研发Copilot

- **编码助手**：减少重复编码工作，提升开发效率和代码质量，助力工程师专注核心逻辑设计。支持代码问答、纠错、改写、生成、注释、解释、优化；覆盖MaxCompute、Hive、MySQL、Oracle、StarRocks等引擎；支持编辑器内弹框提问、推测修改内容提示、快捷键操作、上下游关联表多文件批量增删改。

- **运维助手**：降低人工排查成本，保障系统高可用性与业务连续性。支持任务及实例概览、失败任务查看、关注任务实例、任务状态查询、智能告警分析、查询表产出任务；提供智能诊断、智能修复及优化、智能日志总结。支持钉钉微应用、企微、飞书等多端扩展与消息通知，支持私有部署。

#### 2. X-数据工程（AutoETL）
自动识别已有资产，避免重复建设，流程驱动、AI提效、降低门槛。

流程：用户上传需求（业务说明书、需求说明书、业务主题、场景等）→ AutoETL Agent分析并推荐生成概念模型 → 用户检查确认 → 自动推荐来源表并生成逻辑模型字段列表 → 根据来源表和逻辑表字段生成计算代码 → 自动识别数据间依赖关系和时效性要求配置调度并上线。

**vs 编码助手**：编码助手是研发人员的智能助手，聚焦编辑器内搜索、配置和重复性代码生成；数据工程是跨全产品功能的一站式智能研发工作台，由AI驱动全流程，用户利用专业知识进行确认和修改。

### 二、智能治理
自动化治理规则生成，智能化根因分析，让治理更高效。治理效率提升50%。

#### 1. X-数据标准

- **智能标准提取**：结合数据探查能力与AI数据识别能力，智能检测词根、码表、标准，生成待选项供用户介入确认，减少手工操作，提升标准制定效率。

- **智能落标**：基于大语言模型，根据输入表元数据及表采样数据智能识别表字段的数据分类。可配置识别范围（仅元数据 或 元数据+采样数据），根据安全规范及大模型部署情况自主配置。

#### 2. X-目录管理

- **元数据自动补全（手动）**：一键智能生成资产基础属性——表：所属目录、资产标签、展示名、描述；字段：展示名、描述。支持自定义指标识别与推荐、关联实体与统计周期推荐、指标目录及标签信息生成。支持快捷修改后一键应用、重新生成。

- **元数据自动补全（批量）**：批量全选资产，后台运行不干扰前台，生成结果经人工确认后应用上线，也可手动修正或补充其他属性信息，配合规则式上架规则灵活智能补全。

#### 3. X-数据质量

- **自动规则创建**：根据表元数据信息，让大模型理解后推荐质量规则，人工确认或修改。（规划中）

- **质量溯源及根因分析**：利用大模型分析问题，提供血缘、相关代码等元数据信息，对数据资产实时探查，侦查问题来源及可能原因，呈现问题原因、可能性及根因。结合大模型通识能力、血缘自动检测、自动探查数据。（开发中，预计V5.4支持）

#### 4. X-数据安全

- **智能生成识别特征**：基于大语言模型，根据特征名称智能生成可表达数据特征的正则表达式，并给出解释，帮助判断配置是否正确。元数据和数据均无需给到大模型。

- **智能识别敏感数据**：基于大语言模型，根据表元数据及采样数据智能识别表字段的数据分类分级。流程：识别关键特征（LLM）→ 查找相关分类分级（RAG）→ 识别字段关联的分类分级（LLM）。可人工审核，确认后生效。

### 三、智能用数
打通数据民主化的"最后一公里"。减少80%低效取数需求，释放IT资源。

#### 1. X-分析（自然语言驱动分析）

- **目标用户**：高级分析师、高级运营、技术人员（有业务知识背景，无需掌握SQL，具备少量SQL阅读能力）。

- **核心能力**：通过自然语言进行取数、分析，生成SQL（支持多表关联、嵌套查询、聚合函数、多条件筛选），可在生成SQL基础上二次加工，支持Excel/CSV导出及简单可视化展示。支持智能纠错、适配多引擎语法、长记忆及多轮对话。

- **技术策略**：基于语义知识库构建（标准、码表、词根、指标库、实体关系、规范化资产、标签、资产目录、词条、企业知识库），实时检索知识库，动态上下文注入，动态规划，自动沉淀高质量SQL样例。

- **vs 小Q·问数**：X-分析面向高级分析师/运营和技术人员，使用NL2SQL2Data技术路线，基于通用基础大模型，直接在Dataphin资产消费模块中使用；小Q·问数面向业务运营/非技术人员，使用NL2DSL2Data技术路线，基于微调的专有模型，对接QBI SQL数据集。

#### 2. X-应用创作
用自然语言描述需求，Agent自动进行需求的产品设计、技术设计和代码实现，自动查找可用数据，基于数据服务API保障数据准确度与安全，最终生成可交互、可操作的数据应用。大模型生成代码（Vibe Coding）。（开发中，预计V5.4可Demo）

### 四、开放集成
深度集成到企业个性化工作流。以MCP Server形式将数据研发、调度、治理、数据服务等核心功能开放，客户可嵌入到自有系统，快速构建数据驱动的业务应用。

#### 1. MCP Server 架构
提供一套Dataphin MCP Server，开放NL2SQL、数据服务、OpenAPI、JDBC Driver的能力，供企业构建个性化Agent。仅使用基础模型，无需训练，不含模型部署服务。

#### 2. DataService MCP Server
基于Dataphin数据服务API构建，支持查询API可用性、获取客户信息、查询交易流水等操作。已上线百炼，提供111个MCP Tools。支持Cline（VSCode）用于对客演示。

#### 3. NL2SQL MCP Server
开放NL2SQL Agent Server，降低数据服务业务场景的难度。（TBD，待百炼升级）

## 技术架构

Dataphin Agents基于角色（Role）和规划器（Planner）的Agent架构：
- **Planner（规划）**：任务拆解、语义解析、关键词扩充、问题改写
- **知识库**：词条、业务知识、码表/维值、模型知识、资产检索
- **SQL生成**：找表、SQL生成，支持MaxCompute等引擎
- 支持qwen_max等模型配置
